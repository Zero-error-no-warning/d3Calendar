# agent.md

このファイルは、このリポジトリで作業するエージェント向けの**確定運用ルール**です。
以降の変更・コミット・PR作成時は、以下を必ず守ってください。

## 疑義管理フロー（必須）

1. 検討中の論点・未確定事項は、`agent.md` には直接書かず、**疑義メモ**を別ファイルで作成して管理する。
2. 疑義が解消したら、疑義メモの内容を根拠に、確定したルールだけを `agent.md` に反映する。
3. `agent.md` には「疑義」「検討中」「未決」状態を残さない。

## 必須ルール

1. **PRの言語**
   - Pull Request の**タイトル**と**本文**は必ず日本語で記載すること。
   - 英語のみのPRは不可。

2. **実装とドキュメントの同期**
   - 機能追加・仕様変更を行った場合、実装だけでなく以下も同時に更新すること。
     - `README.md`
     - `examples/` 配下のサンプル
     - `docs/` 配下の公開デモ/ドキュメント

3. **表示仕様の優先順位（カレンダー表示）**
   - `view.span` と `aroundDays` が同時指定された場合は、**`aroundDays` を優先**すること。

4. **初版のスコープ**
   - 繰り返し予定（RRULE 等）や厳密なタイムゾーン処理は初版スコープ外。
   - 初版は単発イベント中心で扱う。

## 補足方針

- Outlook 完全一致ではなく、「Outlook 風の操作感」を重視する。
- 最小実装を維持しつつ、利用者が動作確認できるサンプルを優先する。
